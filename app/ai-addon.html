How to use

Run start_voicegen_ai_addon.ps1, keep VoiceGen on port 3113, then tell agents to use the port 3114 gateway.

Gateway calls are visible immediately. Direct calls to port 3113 only appear after completion through generated outputs.

AI Voice Guide

AI note: When the user asks for follow-up voice lines for the same character, keep the same core voice design prompt from the previous successful take. Do not replace it with a new shorthand description.

For a gradual tone change, copy the prior character prompt and make small targeted edits only. Example: change a tiny-chibi fashion-spark mascot with a bright, glossy, upbeat voice into a tiny-chibi fashion-spark mascot with a somewhat bright, glossy, upbeat voice when the user wants the bright tone softened while preserving Marina as the same character.

For Marina specifically, preserve the tiny-chibi fashion-spark mascot identity unless the user asks for a different character. Add darker, softer, slower, brighter, or more dramatic direction as modifiers to the same base voice design rather than starting over.

PowerShell binary response caveat: PowerShell Invoke-WebRequest can fail locally with a NullReferenceException while handling the binary WAV response from /api/generate. If that happens, do not assume generation failed. Check the addon event log and the outputs folder, then use a binary-safe client such as System.Net.Http.HttpClient and write bytes with [System.IO.File]::WriteAllBytes(...).

PowerShell heredoc caveat: PowerShell does not support Bash-style python - <<'PY' heredocs. For quick Python probes such as WAV duration checks, use python -c "...", a temporary script file, or a PowerShell-native command shape instead.
